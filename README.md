# 2026.1-IOT-Aplicada-A-Conservacao-Marinha
IoT Aplicada à Conservação Marinha: Prototipagem de um Sistema de Monitoramento de pH para Prevenção do Branqueamento de Corais

### Coisas que precisam ter no Repositório
O repositório de dados deverá ser disponibilizado em uma plataforma pública de compartilhamento de
código (como GitHub, por exemplo), e conter os seguintes elementos:
* Descrição do projeto;
* Código fonte do dispositivo;
* Código fonte da plataforma de programação low code;
* O endereço eletrônico (link) do vídeo no YouTube.

As informações disponibilizadas no repositório devem permitir que um terceiro consiga
reproduzir o projeto na íntegra

Cuidado para não expor dados sensíveis, como números de telefone e chaves (token) de API.
Neste caso tais dados devem ser anonimizados.

O repositório não deve conter arquivos compactados, sob pena de não serem avaliados.


### Coisas que precisam ter no Vídeo
O vídeo com a demonstração do funcionamento deverá conter os seguintes elementos:
* Título do projeto, nome dos integrantes do grupo e a data de criação;
* Descrição geral do projeto, incluindo o contexto, a motivação e o problema que se deseja solucionar,
bem como qual ODS da Agenda 2030 da ONU será atendida;
* Funcionamento do(s) sensor(es), incluindo suas adaptações, quando for o caso;
* Funcionamento do(s) atuador(es), quando for o caso, incluindo suas adaptações;
* Integração com a(s) API(s);
* Integração com o banco de dados;
* Demonstração do(s) dashboard(s).
Se o vídeo for produzido a partir de um smartphone, ele deve ser feito com o aparelho na
posição paisagem. O vídeo deve conter narração e ter uma duração de no máximo 5 ou 6
minutos.

### Critérios de avaliação:
Na segunda entrega, referente a nota intermediária N2, serão adotadas as seguintes rubricas de
avaliação do artigo completo, com todas as seções, o protótipo, o repositório de dados e o vídeo com a
demonstração do seu funcionamento:
* Requisitos avaliados no artigo (até 5,0 pontos):
  * [1 ponto] Correção dos itens faltosos da primeira entrega e adição do abstract e resumo;
  * [2 pontos] Materiais e métodos;
  * [1 ponto] Resultados;
  * [0,5 ponto] Conclusões;
  * [0,5 ponto] Referências.

* Requisitos avaliados na apresentação e implementação do projeto final (até 5,0 pontos):
  * [3 pontos] Vídeo compartilhado no YouTube com a apresentação e demonstração do
protótipo. O vídeo deverá demonstrar o funcionamento do sensor e/ou atuador, o uso do
protocolo MQTT, a integração do ambiente de programação low code com a(s) API(s) e o
bando de dados e a visualização das informações no(s) dashboard(s);
  * [2 pontos] Disponibilização da descrição do projeto, dos códigos fonte do dispositivo e da
plataforma de programação low code no formato JSON (JavaScript Object Notation) e do
endereço eletrônico do vídeo em um repositório público.

Na segunda entrega, poderão ainda serem aplicadas as seguintes penalidades:

* Menos 1 ponto se não forem utilizados dois ou mais dispositivos simulados, no caso de estarem
substituindo o(s) disposto(s) físico, ou quando não estiverem atendendo aos requisitos descritos
neste documento;
* Menos 2 pontos se não utilizar o protocolo MQTT;
* Menos 2 pontos se não utilizar o Node-RED ou similar;
* Menos 1 ponto se não integrar com pelo menos uma API;
* Menos 2 pontos se não integrar com pelo menos um banco de dados em nuvem;
* Menos 2 pontos se não criar os dashboards de visualização.

A média final será calculada tendo como base 30% da nota da primeira entrega mais 70% da nota da
segunda entrega.
